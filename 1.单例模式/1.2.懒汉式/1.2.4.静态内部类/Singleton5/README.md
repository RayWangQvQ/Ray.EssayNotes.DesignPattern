# 静态内部类懒汉式

## 优点

* 懒加载

内部类和函数相同，只有在被使用到的时候，才会去执行。

在类被装载时，不会去执行内部类的代码，只有当首次调用 GetInstance() 方法时才会去执行内部类并实例化单例对象。

* 线程安全

利用了CLR实例化类的机制。

* 避免效率浪费

## 缺点

暂无

## 思考

内部类必须是静态类吗？不是静态可不可以？

答：可以的，但最好保证该类内部都是静态成员

原因：

* 不管是不是静态，都是懒加载（没用到的时候不会执行内部类的代码）
* 假如不是静态，只是 private class ，但是因为该内部类只有一个静态字段，永远不会被实例化，其实还是相当与静态类

## 总结

推荐使用。
