# 构建器模式

当待构建的Product比较复杂，内部包含多个组成部件的复杂对象，往往使用Builder模式，利用构建器构建其部件。

## 结构

* 产品：往往内部组成复杂
* 抽象构建器（IBuiler）：一般是接口/抽象类，抽象出构建产品（内部零件）的步骤
* 具体构建器：实现 IBuilder 接口，完成复杂产品的各个部件的具体创建方法
* 指挥者（Director）：它调用建造者对象中的部件构造与装配方法完成复杂对象的创建，在指挥者中不涉及具体产品的信息

通常也可以没有指挥者，因为调用IBuilder的人自然就是指挥者，在本例里AppService就充当了指挥者的角色。

## 与工厂模式比较

* 工厂模式主要关注对同一系列的产品的创建，也就是产品往往有继承关系
* 构建器模式主要关注于怎么创建一个内部组成复杂的类

## 说明

构建器模式在.NET Core框架里使用非常广泛。

该Sample里IBuilder的BuildXXX方法不是返回void而是返回IBuilder本身的做法，就是.NET Core框架里的常见用法，这样可以在构建产品的时候可以连续点出多个构建方法（如AppService里的写法）。
